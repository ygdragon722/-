# 通关读感审稿 · 2026-06-21

> 范围：按当前三天探案 VN 主流程通读开场、Day1 三场、Day2 两场、Day3 双抉择和终局判词。目标不是再扩内容，而是判断哪里还需要发布前的剪辑和文字微调。

## 结论

项目已经进入“可发布前打磨”阶段，不建议继续新增人物、支线、系统或大段剧情。主链条成立：现代读者自以为懂红楼，入梦成为宝玉，第一天读人，第二天看见体面吞人，第三天被迫在“玉”和“无名女孩”之间做选择，终局再反过来读玩家。

当前最值得做的是小修，不是扩写。

## 已直接微修

1. 第三天过渡原文说“井边那个女孩，到底是怎么没的”，但后续玩家实际选择的是“记不记得她”，不是侦破死因。因此改为“井边那个女孩，还要不要有人记得”。
2. Day2 总结里“只有你，记得井边那个名字”改为“井边那个女孩”，避免和终局“无名/没有册子”主题打架。
3. 抉择二铺陈里“没有姓名”改为“没有正经写下的姓名”，保留“小蝉”这个可被玩家记住的称呼，同时强调她没有被正式记录。

## 强项

- 开场“你读了十年红楼 / 你尤其懂宝玉”是很好的玩家画像，支撑终局“反过来读你”。
- Day1 三场分工清楚：凤姐是撑场和账，黛玉是玉与愿望，王夫人是“拾干净”的冷伏笔。
- Day2 明显升级：贾母负责“不看”，王夫人佛堂负责“自认慈悲地沾手”。这一天已经不再是解谜，而是道德恐怖。
- 井边遗物和女孩判词是全剧情绪中心，方向正确，应该继续保护这条线。
- 终局判词机制已经有传播点，“反过来读你”的标题和诗句有截图价值。

## 还可打磨

### P1 · 终局节奏再停一口气

现在终局已能走通，也有压轴停顿。但读感上，“她的判词”之后到“反过来读你”仍然略像同一套翻页流程。发布前可以考虑让女孩判词之后多一个极短空拍，或者在进入玩家判词前加一句过桥文案，例如“然后，镜子转向了你。”

目标：让玩家感觉“我刚刚审判了世界，现在轮到我被看见”。

### P1 · 选择页的两个按钮文案可以更锋利

“让玉回来 / 让它继续藏着”已经够清楚；“为她写下 / 转身离开”也成立。但第二个选择若想更刺痛，可以把“转身离开”再具体一点，比如“就当没看见”或“把她留在井边”。这会更狠，但也更不留余地，需要用户口味确认。

建议暂不直接改，人工玩一遍后再定。

### P2 · 第一场凤姐略长，但可保留

凤姐共情路线的回应信息量最大，也承担第一场建立“读人玩法”的任务。它长，但目前不算拖。如果发布后感觉前十分钟太密，可以先剪凤姐共情 outcome 的中段，不动后面几场。

### P2 · README 和开发计划定位仍有历史痕迹

README 已改成当前形态，但 `docs/红楼探案_开发计划.md` 仍写“2 小时纯点击互动红楼电影”和若干较大的开发轨道。它作为历史计划没问题，但如果要给未来接手者看，建议再加一个“当前封版目标：15 分钟短篇版”的顶部注记。

## 不建议做

- 不要再加宝钗、元妃、晴雯等新人物线。
- 不要再做新的系统层，例如线索板、镜子账本、任务簿，除非当前短篇已经发布并确认值得扩。
- 不要把“小蝉之死”写成侦探式真相揭露。现在更强的点是“她为什么不被记得”，不是“凶手是谁”。
- 不要把结局做成好坏判定。现在的痛感来自每条路都有代价。

## 下一步建议

1. 人工手机通关一次，专门看两个选择是否够重。
2. 若觉得终局还差一口气，优先加“镜子转向你”的过桥拍。
3. 若觉得女孩线还不够刺痛，再讨论是否把“转身离开”换成更锋利的按钮文案。
4. 然后进入发布准备：提交当前改动，部署 GitHub Pages，检查线上链接。
