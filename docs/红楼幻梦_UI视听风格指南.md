# 红楼幻梦 UI/视听风格指南

## 定位

《红楼幻梦》不是普通网页故事，而是竖屏短篇视觉小说。第一屏必须先让玩家知道“游戏开始了”，再进入现代大观园剧情。

核心风格：镜中旧梦、夜色大观园、铜镜、工笔水墨、暗金、墨黑、旧朱红、慢节奏。

## 第一屏

- 使用正式标题页，而不是直接进入“女生进大观园”。
- 标题页结构：背景主视觉、游戏名、短副标题、开始/继续/音量入口。
- 标题页之后先给一张“开场牌”，再进入正文冷开场。开场牌负责明确玩家身份、地点和游玩状态，例如“现代女性玩家 / 城郊大观园景区 / 游园开始”，避免第一句剧情显得突兀。
- 报名字页属于“入梦确认”，不是普通表单。它应使用章标、墨色面板、统一输入框和主按钮，视觉上像玩家在镜前留下名字。
- 推荐主视觉：旧铜镜放在朱漆案上，镜中映出夜色大观园、灯火、月影和风月鉴。
- 标题：`红楼幻梦`
- 副标题：`一场读人，也被人读的梦`
- 主要按钮：`开始入梦`
- 有存档时显示：`继续旧梦`

## UI 组件

统一使用“墨色半透明框 + 暗金/白色细边 + 轻微磨砂”的语言。

- 圆角：6px 到 8px，除非特殊装饰，不再混用大圆角。
- 字幕框：深墨背景、细边框、底部提示淡金/淡灰。
- 字幕分段：在 `SubtitleBox` 中只有空行才切换下一张字幕卡；普通换行只用于同一卡内排版，避免玩家为一句话点太多次。
- 选项框：和字幕框同源，hover 时边框变亮，必须 `cursor-pointer`。
- 回退按钮：小型墨色框，文案统一为“上一句 / 回看上一段 / 上一幕”。
- 场景名牌：统一走 `ScenePlaque`。剧情地点用左上窄竖金线 + 墨色底；章节/抉择/终局用顶部居中的小型墨色章标。
- 开始/继续按钮：暗金边框，hover 有微弱金色背景。
- 普通按钮统一走 `VNButton`，按 `primary / secondary / quiet / ghost` 分级；特殊抉择按钮可保留局部状态样式，但必须遵守同一圆角、墨色底、边框、hover 和手型规则。

## 字体与文字

- 标题：宋体/明体气质，字距靠布局 gap，不用负字距。
- 正文：`Noto Serif SC`, fallback `Songti SC`, serif。
- 字幕正文保持 16px 左右，移动端优先可读。
- 提示文字小而淡，不和正文争抢。

## 颜色

- 背景主色：墨黑、深褐、夜色蓝黑。
- 强调色：暗金、旧金、低饱和朱红。
- 文本色：米白、淡金、灰白。
- 禁用：霓虹、高饱和紫蓝渐变、现代手游亮色按钮、Y2K 金属泡泡感。
- 画面亮度优先在渲染层统一调整，不优先批量改原图。当前共享规则为 `.vn-scene-image`，用 `brightness(1.18) contrast(1.04) saturate(1.04)` 轻度提亮，再按场景降低遮罩强度。

## 动效

- 进入：慢淡入、轻微上浮、镜光扫过。
- 字幕：先让画面出现，再显示字幕框。
- 标题：逐字浮现，模糊到清晰，避免炫技。
- hover：150-300ms，边框/背景变化，不依赖 hover 传达关键操作。
- 必须支持 `prefers-reduced-motion`。

## 声音

先做轻量声音层，不做复杂配乐系统。

- 标题页：低音古琴、远处钟磬、夜风或园林环境声。
- 点击继续：轻纸页/玉佩/木质轻响。
- 选择：短促铜镜或玉石声。
- 关键转场：极轻镜面低鸣。
- 必须提供静音/音量入口，默认音量低。
- 推荐实现：小型 `sound.ts` 或 howler.js，不要散落多个 `new Audio()`。
- 当前轻量实现采用 `sound.ts` 的 Web Audio 合成短音，不新增素材文件；后续如需要更有质感，再替换为真实音频素材或 howler.js。

## 美术生成提示词骨架

竖屏 9:16，古典中国宅院夜景，旧铜镜，风月鉴，镜中映出大观园灯火与月影，工笔水墨融合，低饱和暗金、墨黑、旧朱红，细腻纸本质感，留出中央标题空间，神秘、古典、梦境感，不要现代霓虹，不要赛璐璐动画线稿，不要西式奇幻 UI。

## 验收

- 375px/390px/440px 竖屏截图。
- 标题页、字幕页、选项页、人物近景、终局页各取一张。
- 检查：人物是否出现、按钮是否像按钮、hover/cursor 是否明确、文字是否溢出、图片是否被裁坏。
- `npm run build` 必须通过。
