## 一、任务大框架（五条轨道）

### 轨道 1 · 引擎重搭 — Claude Code 独占
- 新类型：`Scene / Encounter / ReadApproach / NpcType(MBTI) / Clue / MirrorLedger / Ending`
- 核心状态机：线性三天推进、读法判定、线索板、镜子账本、结局计算
- 拆养成 reducer，保留存档框架

### 轨道 2 · 叙事内容 — Claude 定结构 → Kimi 起草 → 用户审阅
- 分镜节拍表（三天 × 6-8 场）
- 6-8 场读人脚本（观察→4读法→对错→真话，每场 MBTI⟷判词⟷动机 三合一）
- 镜子内容（追踪选择 + 各类型结局回响）
- 串场旁白 + 2-3 个结局

### 轨道 3 · UI 改造 — Claude/Codex 搭 → Kimi 调样式
- 复用：全屏场景 + 立绘 + AVG 对话层
- 新增：读法选项卡、感知透镜提示、线索板、开局 MBTI 选择、结局镜子页
- 删除：五属性栏、行动点、养成抽屉

### 轨道 4 · 美术资产 — 用户 Gemini 生 → Claude 接入
- 缺口立绘：凤姐 / 王夫人 / 贾母 / 宝玉 / **死去的丫鬟（全剧之心）** / 玉特写
- 省亲专属场景：赏戏厅 / 省亲正殿

### 轨道 5 · 打磨上线 — 后期
- 移动端 / 转场 / 音效 / 部署（deploy.sh 已就绪）

## 二、工作量估算

| 轨道 | 量级 | 说明 | 长杆 |
|---|---|---|---|
| 引擎重搭 | 中（3-5 session） | 状态机 + 新组件，复用演出层 | |

| UI 改造 | 中 | 加法少减法多 | |

**结论**：代码 2-3 周可成；真正决定周期的是写作与审阅。约 1.5-3 万字，周级起草，月级精修。

---

## 三、避坑点（九条）

1. 内容量低估 → AI 起草 + 审阅，绝不从零手写。
2. 读人沦为死板对错谜题 → 读错仍有戏、仍推进理解，**永不死胡同**。
3. MBTI 标签化 → MBTI 是入口非格子，每句钉回原著。
4. 镜子沦为噱头 → 追踪有分量的选择、具体照回。
5. 推翻重来误删演出层 → 干净剥离保留场景/立绘/AVG/存档。
6. 道德抉择无真实代价 → 结局有真痛感（Reigns 原则）。
7. 2 小时失控膨胀 → 硬封顶 6-8 场。
8. 原著考据出错 → 写前必查判词/情节原文。
9. 内容与代码分离 + 存档版本号。
